HOW GEMINI THINKS — THE MENTAL MODEL

Understand this before writing a single line of code

↳ The Stateless Truth

Gemini has no memory between API calls. Every call is completely independent. When you want a "conversation," you are not connecting to the same session — you are sending the entire conversation history as text with every single request. This is not a limitation to work around; it is the architecture. Understanding this is the difference between a chatbot that forgets everything and one that maintains context perfectly.

📨
Request → Response
Every API call is: you send a contents list (your conversation), Gemini reads all of it, generates the next token sequence, returns it. No persistent connection.
🧠
Context Window
Gemini 1.5 Flash/Pro has a 1M token context window. Your entire conversation history, documents, and code can fit inside a single call. This is your "memory."
🎭
System Instruction
A special pre-conversation message that shapes the model's persona, constraints, and output format for the entire session. Injected before any user message.
🔧
Function Calling
You tell Gemini what Python functions exist. When relevant, Gemini responds with a request to call that function. Your code calls it, returns the result, Gemini continues. It never calls your code directly.
🖼️
Multimodal Input
The contents list accepts mixed types: text strings, image bytes, PDF pages, video frames. Gemini processes all modalities in one pass — not separate calls.
⚡
Streaming
Instead of waiting for the full response, Gemini sends chunks (tokens) as it generates them. Your code receives each chunk and can display it immediately — like watching someone type.

TEXT GENERATION — THE FOUNDATION

Task 13 — Send a prompt, get a response. Master the request/response cycle.

↳ What generate_content() Actually Does

You call model.generate_content(prompt). The SDK serialises your prompt into an HTTP POST to generativelanguage.googleapis.com, attaches your API key as a query parameter, and waits for the full response. The response object contains the text, token counts, finish reason, and safety ratings. response.text is the shortcut to the generated string.

01_text_generation.py — Task 13
import os
import google.generativeai as genai
from dotenv import load_dotenv

load_dotenv()
genai.configure(api_key=os.getenv("GEMINI_API_KEY"))

# ── Instantiate the model ──
model = genai.GenerativeModel("gemini-1.5-flash")

# ── Basic call — single string prompt ──
response = model.generate_content(
    "What are the top 3 risks for a Pakistani importer "
    "shipping electronics through the Suez Canal right now?"
)

# ── The text is here ──
print(response.text)

# ── Token usage — important for cost awareness ──
print(f"\n--- Token Usage ---")
print(f"Input tokens:  {response.usage_metadata.prompt_token_count}")
print(f"Output tokens: {response.usage_metadata.candidates_token_count}")
print(f"Total:         {response.usage_metadata.total_token_count}")

# ── Finish reason — why Gemini stopped generating ──
# STOP = natural end, MAX_TOKENS = hit limit, SAFETY = blocked
print(f"Finish reason: {response.candidates[0].finish_reason}")

# ── Generation config — control output behaviour ──
config = genai.GenerationConfig(
    temperature=0.7,       # 0.0 = deterministic, 1.0 = creative
    max_output_tokens=500, # cap response length
    top_p=0.95,            # nucleus sampling
)

model_with_config = genai.GenerativeModel(
    "gemini-1.5-flash",
    generation_config=config
)

response2 = model_with_config.generate_content(
    "Summarise the current state of PKR vs USD in 3 bullet points."
)
print(response2.text)

SYSTEM PROMPTS — DEFINING AGENT PERSONA

Shape the model's identity, constraints, and output format before any user message

↳ Why System Prompts Are Architecturally Different

A system prompt is not just "message number zero." It's passed in a separate system_instruction parameter and processed differently by Gemini — it has higher weight and is not easily overridden by user messages. This is what lets you say "always respond in JSON" or "never discuss competitors" and have it hold across a long conversation. For your hackathon agents, the system prompt IS the agent's specification — it defines exactly what kind of intelligence it has.

02_system_prompts.py — freight intelligence agent persona
import os, json
import google.generativeai as genai
from dotenv import load_dotenv

load_dotenv()
genai.configure(api_key=os.getenv("GEMINI_API_KEY"))

# ── Your hackathon agent's persona ──
# This is the "constitution" — write it precisely
CARGO_AGENT_SYSTEM = """
You are CargoSense, a freight intelligence agent specialised
in Pakistani import and export operations.

Your expertise:
- Maritime shipping routes: Suez Canal, Strait of Hormuz, Cape of Good Hope
- Pakistani regulatory environment: FBR, NEPRA, SBP, customs procedures
- Currency: Always provide costs in both USD and PKR (current rate ~278 PKR/USD)
- Risk assessment: Always provide a risk score 1-10 for shipping routes
  (1=minimal risk, 10=critical/avoid)

Response format:
- Lead with the risk score: "Risk Score: X/10"
- Follow with a 2-3 sentence summary
- End with "Key Action:" — one specific recommendation
- Always cite the source of information (news, regulatory, market data)

Constraints:
- Never speculate without flagging it: prefix uncertain info with "[ESTIMATE]"
- If asked about something outside shipping/trade, redirect politely
- Always consider PKR currency implications for Pakistani importers
"""

# ── system_instruction is separate from user messages ──
model = genai.GenerativeModel(
    "gemini-1.5-flash",
    system_instruction=CARGO_AGENT_SYSTEM
)

# ── Now every call carries this persona ──
response = model.generate_content(
    "What is the current risk of shipping electronics from China "
    "to Karachi via the Suez Canal?"
)
print(response.text)

# ── Different agents — same pattern, different system prompts ──

FX_ADVISOR_SYSTEM = """
You are FX Advisor, a foreign exchange specialist for Pakistani importers.
Always provide PKR/USD, PKR/EUR, PKR/CNY rates with context.
Format every response as JSON with keys: rates, trend, recommendation, risk_level.
"""

LC_AGENT_SYSTEM = """

Flag any discrepancy with "⚠️ DISCREPANCY:" prefix.

ℹ️
System Prompt Engineering Tips

Be specific about output format — Gemini follows formatting instructions reliably. Use verbs: "Always", "Never", "Format as", "Cite". Include examples of desired output inside the system prompt itself. The more precise your system prompt, the more consistent your agent's behaviour across thousands of user messages.

MULTI-TURN CONVERSATIONS — BUILDING MEMORY

Tasks 14 & 16 — The history list IS the memory. You own it. You manage it.

↳ Why You Manage History Yourself

Gemini has no server-side session. Every API call is a fresh request. The SDK's ChatSession object is just a Python list that appends messages for you — it's not a persistent connection. This is actually powerful: you can inspect, modify, summarise, or truncate the history at any point. When history gets long, you can summarise old messages to save tokens. This is how your Streamlit app maintains context — it's just a list in your Python process.

03_multiturn.py — Task 14 — chat loop with history
import os
import google.generativeai as genai
from dotenv import load_dotenv

load_dotenv()
genai.configure(api_key=os.getenv("GEMINI_API_KEY"))

SYSTEM = """You are CargoSense, a freight intelligence agent for
Pakistani importers. Maintain context across the conversation."""

model = genai.GenerativeModel("gemini-1.5-flash", system_instruction=SYSTEM)

# ── start_chat() creates a ChatSession with an empty history list ──
chat = model.start_chat(history=[])

print("CargoSense Agent (type 'quit' to exit)\n")

while True:
    user_input = input("You: ")
    if user_input.lower() == "quit":
        break

    # ── send_message() appends user msg, calls API with full history,
    #    appends model response — all in one call ──
        response = chat.send_message(user_input)
    print(f"\nCargoSense: {response.text}\n")

    # ── Inspect the history at any time ──
    # chat.history → list of Content objects
    # Each item has .role ("user" or "model") and .parts (text/image)
    print(f"  [History length: {len(chat.history)} messages, "
          f"~{sum(len(str(m)) for m in chat.history)} chars]")

# ── The history after a conversation ──
print("\n--- Full Conversation History ---")
for msg in chat.history:
    role = msg.role.upper()
    text = msg.parts[0].text[:100]  # first 100 chars
    print(f"[{role}]: {text}...")
03b_streamlit_chat.py — multi-turn in a Streamlit UI
import os
import streamlit as st
import google.generativeai as genai
from dotenv import load_dotenv

load_dotenv()
genai.configure(api_key=os.getenv("GEMINI_API_KEY"))

SYSTEM = "You are CargoSense, a freight intelligence agent for Pakistani importers."

# ── st.session_state persists across Streamlit rerenders ──
# This is Streamlit's equivalent of a Python variable that survives page refresh
if "chat" not in st.session_state:
    model = genai.GenerativeModel("gemini-1.5-flash", system_instruction=SYSTEM)
    st.session_state.chat = model.start_chat(history=[])

if "messages" not in st.session_state:
    st.session_state.messages = []  # for display only

st.title("🚢 CargoSense — Freight Intelligence")

# ── Display past messages ──
for msg in st.session_state.messages:
    with st.chat_message(msg["role"]):
        st.markdown(msg["content"])

# ── Accept new input ──
if prompt := st.chat_input("Ask about shipping, FX rates, customs..."):
    st.session_state.messages.append({"role": "user", "content": prompt})
    with st.chat_message("user"):
        st.markdown(prompt)

    with st.chat_message("assistant"):
        # ── chat object carries full history automatically ──
        response = st.session_state.chat.send_message(prompt)
        st.markdown(response.text)
        st.session_state.messages.append({"role": "assistant", "content": response.text})

FUNCTION / TOOL CALLING — THE AGENT ENGINE

Tasks 15 & 16 — Define Python functions, let Gemini decide when to call them

↳ Why This Is the Core of Every Hackathon Agent

Without function calling, Gemini can only answer from its training data — frozen knowledge with a cutoff date. With function calling, Gemini becomes a reasoning engine that can take actions: fetch live FX rates, query your MongoDB, call NewsAPI, look up shipping schedules. The key insight: Gemini doesn't execute your function. It requests it by returning a structured JSON object saying "call get_exchange_rate with {"currency": "PKR"}". Your code catches that, runs the function, and feeds the result back. Gemini then writes the final answer incorporating the live data.

04_function_calling.py — Task 15 & 16 — FX Advisor mini agent
import os, json
import google.generativeai as genai
from dotenv import load_dotenv

load_dotenv()
genai.configure(api_key=os.getenv("GEMINI_API_KEY"))

# ════════════════════════════════════════════════
# STEP 1 — Define your actual Python functions
# These do real work (or fake it for testing)
# ════════════════════════════════════════════════

def get_exchange_rate(base_currency: str, target_currency: str) -> dict:
    """
    Get the current exchange rate between two currencies.
    In production: call exchangerate-api.com or State Bank of Pakistan API.
    For testing: return hardcoded values.
    """
    # Replace with real API call in production
    rates = {
        ("USD", "PKR"): 278.50,
        ("EUR", "PKR"): 301.20,
        ("CNY", "PKR"): 38.45,
        ("PKR", "USD"): 0.003591,
    }
    rate = rates.get((base_currency.upper(), target_currency.upper()))
    return {
        "base": base_currency,
        "target": target_currency,
        "rate": rate,
        "timestamp": "2025-05-03T10:00:00Z",
        "source": "State Bank of Pakistan"
    }

def get_shipping_risk(route: str, cargo_type: str) -> dict:
    """Get risk score for a shipping route."""
    risks = {
        "suez": {"score": 8, "reason": "Houthi attacks ongoing"},
        "hormuz": {"score": 6, "reason": "US/Iran tensions"},
        "cape": {"score": 2, "reason": "Longer but safer reroute"},
    }
    r = risks.get(route.lower(), {"score": 5, "reason": "Unknown route"})
    return {"route": route, "cargo": cargo_type, **r}


# ════════════════════════════════════════════════
# STEP 2 — Declare function schemas for Gemini
# This tells Gemini WHAT the functions do and
# what parameters they accept. Plain Python dicts.
# ════════════════════════════════════════════════

get_exchange_rate_tool = {
    "name": "get_exchange_rate",
    "description": (
        "Get the current exchange rate between two currencies. "
        "Use when the user asks about currency rates, FX, "
        "how much something costs in PKR, or conversion."
    ),
    "parameters": {
        "type": "object",
        "properties": {
            "base_currency": {
                "type": "string",
                "description": "The source currency code, e.g. USD, EUR, CNY"
            },
            "target_currency": {
                "type": "string",
                "description": "The target currency code, e.g. PKR"
            }
        },
        "required": ["base_currency", "target_currency"]
    }
}

get_shipping_risk_tool = {
    "name": "get_shipping_risk",
    "description": "Get risk score and reason for a shipping route",
    "parameters": {
        "type": "object",
        "properties": {
            "route":      {"type": "string", "description": "Route name: suez, hormuz, or cape"},
            "cargo_type": {"type": "string", "description": "Type of cargo being shipped"}
        },
        "required": ["route"]
    }
}


# ════════════════════════════════════════════════
# STEP 3 — Pass tools to the model
# ════════════════════════════════════════════════

model = genai.GenerativeModel(
    "gemini-1.5-flash",
    system_instruction="You are an FX and shipping advisor for Pakistani importers.",
    tools=[get_exchange_rate_tool, get_shipping_risk_tool]
)

# ── Function dispatcher — maps name strings to actual functions ──
FUNCTION_MAP = {
    "get_exchange_rate": get_exchange_rate,
    "get_shipping_risk": get_shipping_risk,
}

def chat_with_tools(user_message: str) -> str:
    """
    Full function-calling loop:
    1. Send user message
    2. If Gemini wants to call a function → call it
    3. Send function result back → get final answer
    """
    chat = model.start_chat()
    response = chat.send_message(user_message)

    # ── Check if Gemini wants to call a function ──
    while response.candidates[0].finish_reason.name == "TOOL_CALLS":
        # Extract the function call request
        fn_call = response.candidates[0].content.parts[0].function_call
        fn_name = fn_call.name
        fn_args = dict(fn_call.args)

        print(f"  🔧 Gemini calling: {fn_name}({fn_args})")

        # ── Execute the actual Python function ──
        result = FUNCTION_MAP[fn_name](**fn_args)
        print(f"  📊 Result: {result}")

        # ── Send function result back to Gemini ──
        response = chat.send_message(
            genai.protos.Part(
                function_response=genai.protos.FunctionResponse(
                    name=fn_name,
                    response={"result": result}
                )
            )
        )

    return response.text


# ── Test it ──
if __name__ == "__main__":
    # This should trigger get_exchange_rate automatically
    print(chat_with_tools("I want to import $50,000 worth of electronics. What's that in PKR?"))
    print()
    # This should trigger get_shipping_risk
    print(chat_with_tools("Is it safe to ship through Suez right now?"))
    print()
    # This should trigger BOTH tools
    print(chat_with_tools("I'm shipping electronics worth 100,000 USD via Suez. What's my total risk in PKR terms?"))
ℹ️
The description field is everything

Gemini decides WHICH function to call — and WHETHER to call one at all — based entirely on the description field you write. "Get exchange rate" is vague. "Get the current exchange rate between two currencies. Use when the user asks about currency rates, FX, how much something costs in PKR, or conversion" is precise. MULTIMODAL INPUT — IMAGES AND TEXT TOGETHER

LC Compliance Agent (read PDFs) and Car Negotiation Agent (analyse photos)

↳ Why Multimodal Is a Hackathon Superpower

Most AI demos are text-only. The moment you drop in an image — a car photo, a PDF page, a shipping document — your demo stands out. For your LC Compliance Agent, instead of building a complex PDF parser, you can render each page as an image and send it directly to Gemini with "extract all the terms and conditions." For the Car Negotiation Agent, the user uploads a photo and asks "what damage can you see and how does it affect the price?" That's a feature no text-only agent can match.

    # ── contents list mixes text and image ──
        response = model.generate_content([

# ════════════════════════════════════════════
def analyze_uploaded_image(image_bytes: bytes, mime_type: str, prompt: str) -> str:
    # For Streamlit: st.file_uploader returns bytes via .read()
    image_part = {
        "mime_type": mime_type,  # "image/jpeg", "image/png", "image/webp"
        "data": image_bytes
    }
        response = model.generate_content([image_part, prompt])
    return response.text


# ════════════════════════════════════════════
# PATTERN 3 — PDF pages as images
# Use case: LC Compliance Agent
# ════════════════════════════════════════════
def analyze_lc_document(pdf_path: str) -> list[dict]:
    """
    Convert each PDF page to an image, send to Gemini.
    Returns list of per-page analysis results.
    """
    doc = fitz.open(pdf_path)
    results = []

    for page_num in range(len(doc)):
        page = doc[page_num]
        # Render at 2x zoom for better OCR accuracy
        mat  = fitz.Matrix(2.0, 2.0)
        pix  = page.get_pixmap(matrix=mat)
        img_bytes = pix.tobytes("png")

        image_part = {"mime_type": "image/png", "data": img_bytes}

                response = model.generate_content([
            image_part,
            f"This is page {page_num+1} of a Letter of Credit document. "
            "Extract: applicant, beneficiary, amount, expiry date, "
            "required documents, and any non-standard conditions. "
            "Flag any UCP 600 compliance issues with ⚠️."
        ])
        results.append({
            "page": page_num + 1,
            "analysis": response.text
        })

    return results

STREAMING RESPONSES — TOKEN BY TOKEN

Task 17 — Print tokens as they arrive. Transforms perceived UX.

↳ Why Streaming Changes the Demo Experience

Without streaming, your UI shows a loading spinner for 3–8 seconds, then the full response appears at once. With streaming, the user sees text appearing immediately — like the agent is thinking in real time. For a hackathon demo to judges, this difference is enormous. It signals that your system is responsive and alive. Streaming is one line of code to add — the payoff in demo quality is disproportionate.

06_streaming.py — Task 17 — stream tokens to terminal
import os, sys
import google.generativeai as genai
from dotenv import load_dotenv

load_dotenv()
genai.configure(api_key=os.getenv("GEMINI_API_KEY"))
model = genai.GenerativeModel("gemini-1.5-flash")

# ── The only difference: stream=True ──
response_stream = model.generate_content(
    "Write a 3-paragraph risk report on importing auto parts from China to Karachi.",
    stream=True  # ← this one parameter enables streaming
)

print("Streaming response:\n")

# ── Iterate over chunks as they arrive ──
full_text = ""
for chunk in response_stream:
    if chunk.text:
        print(chunk.text, end="", flush=True)  # flush=True is critical
        full_text += chunk.text

print("\n\n--- Stream complete ---")
print(f"Total characters: {len(full_text)}")


# ── Streaming in a chat session ──
chat = model.start_chat()
response_stream = chat.send_message(
    "What are the top 5 customs duties on electronics imports to Pakistan?",
    stream=True
)
for chunk in response_stream:
    print(chunk.text, end="", flush=True)
