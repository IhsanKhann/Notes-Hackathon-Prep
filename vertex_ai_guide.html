THE VERTEX AI MENTAL MODEL

What Vertex AI is, how it relates to Gemini API, and how it fits your project

↳ The Core Relationship

Think of it this way: Gemini API Direct is like using a public API with an API key — fast, simple, great for prototyping. Vertex AI is like checking into the same hotel but through the corporate account — same rooms (same Gemini models), but with enterprise billing, access controls, audit logging, and integration with every other GCP service. The models are identical. The authentication and surrounding ecosystem are different.

WHEN TO USE WHICH — THE DECISION GUIDE

Task 22 — A definitive framework for choosing your platform

Use Vertex AI When...
  • App is deployed on Cloud Run / GCP
  • Need IAM / service account authentication
  • Using Agent Builder for no-code agents
  • Need Vertex AI Search for document RAG
  • Production apps post-hackathon
  • Need unified GCP billing
  • Need access to non-Gemini models (Llama, Claude via Model Garden)
  • Compliance requires audit logs
Use Gemini API Direct When...
  • Local development and prototyping
  • Simpler auth — just an API key
  • Quick iteration, testing prompts
  • Hackathon demo build (Day 1–8 hours)
  • Building on a non-GCP platform
  • No billing account set up yet
  • Testing ideas in AI Studio playground
  • Free tier is sufficient for load
↳ Recommended Strategy for Your Hackathon

Development phase: Use Gemini API Direct with your AI Studio key. Build fast, iterate fast, test all your agents and prompts.

Demo deployment: Switch to Vertex AI in your Cloud Run service. Use your service account credentials. Your code changes by exactly 4 lines — the import and the model initialisation. Everything else (prompts, function calling, streaming) works identically.

TASK 20 — SERVICE ACCOUNT AUTHENTICATION

↳ Three Authentication Methods for Vertex AI

Method 1 — Application Default Credentials (ADC): Run gcloud auth application-default login on your local machine. The SDK finds credentials automatically. Best for local development.

Method 2 — Service Account JSON Key: Download a JSON key file, set GOOGLE_APPLICATION_CREDENTIALS env var to its path. Required for Task 20. Works everywhere.

Method 3 — Workload Identity (Cloud Run): No key file. Cloud Run's attached service account provides credentials automatically. This is what your deployed app uses from Guide 1.

POWERSHELL — create service account and download key (Task 20)
# Use the service account created in the Cloud Run guide,
# or create a new one specifically for Vertex AI
$PROJECT_ID = "your-project-id"
$SA_EMAIL   = "project-test-sa@$PROJECT_ID.iam.gserviceaccount.com"

# Grant Vertex AI User role
gcloud projects add-iam-policy-binding $PROJECT_ID `
  --member="serviceAccount:$SA_EMAIL" `
  --role="roles/aiplatform.user"

# Download the JSON key (use ONLY for local dev / Task 20)
gcloud iam service-accounts keys create vertex-sa-key.json `
  --iam-account $SA_EMAIL

# Add to .env (point to the key file)
# GOOGLE_APPLICATION_CREDENTIALS=C:\Users\7480\Desktop\python-codes\Project_test\vertex-sa-key.json

# ⚠️ Add vertex-sa-key.json to .gitignore immediately
⚠️
Never commit the JSON key file to Git

Add vertex-sa-key.json and *.json to your .gitignore right now. If you push it to a public repo, attackers will find it within seconds via GitHub token scanners. For your Cloud Run deployment, you don't need the key file at all — Workload Identity handles it from Guide 1.

VERTEX AI AGENT BUILDER — NO-CODE AGENTS

Task 21 — Create an agent, upload a PDF, test grounded queries in the console

↳ Why Agent Builder Exists and When to Use It

Agent Builder is Google's answer to: "I want a production-grade conversational agent with document search, but I don't want to write the RAG pipeline from scratch." It gives you a managed system that handles chunking, embedding, vector storage, retrieval, and response generation — all configured through a UI. For your hackathon, use it to rapidly prototype the Government Service Navigator or LC Compliance Agent. Upload the official FBR guide or UCP 600 PDF, and you have a searchable knowledge base in under 10 minutes. Then you can call this agent from your FastAPI backend via API.

TASK 21 — CREATE AN AGENT BUILDER AGENT (CONSOLE WALKTHROUGH)

1
Open Agent Builder

Go to GCP Console → Navigation menu → "Vertex AI" → "Agent Builder". Or search "Agent Builder" in the top search bar. First time: it may ask you to enable the Discovery Engine API — approve it.

2
Create an App

Click "Create App" → Choose "Search" (for document search / RAG) or "Conversational Agent" (for a full chatbot). For Task 21, choose "Search". Name it cargo-sense-search.

3
Create a Data Store

Click "Create Data Store" → "Cloud Storage" → upload your PDF (e.g. the FBR Import Procedures guide, or a sample Letter of Credit). Alternatively paste a website URL for web crawling. Agent Builder chunks and embeds the document automatically.

4
Wait for Indexing

Indexing takes 5–15 minutes for a PDF. You'll see the status change from "Pending" to "Active" in the Data Store panel. For a hackathon, upload your PDF the night before.

5
Test a Grounded Query

Go to the "Preview" tab. Type: "What documents are required to import electronics under an LC?" The response will include quotes from your uploaded PDF with citations. This is RAG working.

6
Get the API Integration Details

Go to "Integration" tab. Find your engine_id and project_id. You'll use these to call Agent Builder from Python code (see code block below).

VERTEX AI SEARCH — RAG GROUNDING

Give your agents access to document knowledge — government guides, UCP 600, FBR rules

↳ What RAG Actually Means and Why It Matters for Your Agents

RAG = Retrieval Augmented Generation. Instead of relying on Gemini's training data (which has a knowledge cutoff and no access to your private documents), you: (1) store your documents in a vector database, (2) when a user asks a question, search for the most relevant document chunks, (3) inject those chunks into the Gemini prompt as context, (4) Gemini answers based on the real document content. For your Government Service Navigator, this means uploading SBP, FBR, and SECP official guides — your agent answers from actual government documentation, not from Gemini's general training data. That's the difference between a generic chatbot and a trustworthy compliance tool.

vertex_rag.py — Vertex AI Search + Gemini grounding in Python
import os import vertexai from vertexai.generative_models import ( GenerativeModel, Tool, grounding ) from dotenv import load_dotenv load_dotenv() vertexai.init( project=os.getenv("GOOGLE_CLOUD_PROJECT"), location="us-central1" ) # ── Option A: Ground with Vertex AI Search data store ── # Use when you've uploaded documents to Agent Builder data_store_path = ( f"projects/{os.getenv('GOOGLE_CLOUD_PROJECT')}/" f"locations/global/collections/default_collection/" f"dataStores/cargo-sense-search_XXXX" # your data store ID ) search_tool = Tool.from_retrieval( retrieval=grounding.Retrieval( source=grounding.VertexAISearch(datastore=data_store_path) ) ) model_with_search = GenerativeModel( "gemini-1.5-flash-001", system_instruction="You are an LC compliance expert. Always cite your sources.", tools=[search_tool] ) response = model_with_search.generate_content( "What documents are required for an LC covering textile imports from China to Pakistan under UCP 600?" ) print(response.text) # The response includes grounding citations — links to the actual documents if response.candidates[0].grounding_metadata: print("\n--- Sources ---") for chunk in response.candidates[0].grounding_metadata.grounding_chunks: print(f"- {chunk.retrieved_context.title}: {chunk.retrieved_context.uri}") # ── Option B: Ground with Google Search (live web results) ── # No data store needed — uses live Google Search as context web_search_tool = Tool.from_google_search_retrieval( google_search_retrieval=grounding.GoogleSearchRetrieval() ) model_with_web = GenerativeModel( "gemini-1.5-flash-001", tools=[web_search_tool] ) response = model_with_web.generate_content( "What is the current PKR to USD exchange rate and what are analysts saying about it?" ) print(response.text)

MODEL GARDEN — BEYOND GEMINI

Access Llama, Mistral, Claude, and 50+ other models through one platform

↳ Why Model Garden Matters Post-Hackathon

Model Garden is a unified model marketplace inside Vertex AI. Same service account, same billing, access to models from Google, Meta, Mistral, Anthropic, and others. For your hackathon, you won't need it — Gemini 1.5 Flash handles everything. But for production, it matters: if a specific task (e.g. legal document analysis) performs significantly better with Claude, you can switch without changing your authentication setup, billing account, or deployment infrastructure.
