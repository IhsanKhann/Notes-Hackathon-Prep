01
Install Python 3.11+ & Set Up a Virtual Environment
The first thing you do at every hackathon. Isolates your packages. Never skip this.
💡
Why a virtual environment?

A venv creates an isolated Python runtime. Your hackathon dependencies (fastapi, anthropic, streamlit…) install inside it without touching system Python. Every team member gets the same locked versions.

STEP 1
Check your Python version

You need 3.11 or higher. FastAPI's async performance and modern type hint features require it.

STEP 2
Create the virtual environment

Run python -m venv env in your project root. This creates an env/ folder containing a self-contained Python interpreter.

STEP 3
Activate it

On Mac/Linux: source env/bin/activate — On Windows: env\Scripts\activate. Your terminal prefix changes to (env).

STEP 4
Install your hackathon stack

Paste the one-liner from your master plan. Every package goes into env/, not system Python.

STEP 5
Freeze your requirements

Run pip freeze > requirements.txt so teammates can reproduce the exact environment with pip install -r requirements.txt.


TERMINAL — full setup sequence
# 1. Confirm Python version (must be 3.11+)
python3 --version

# 2. Create virtual environment in current folder
python3 -m venv env

# 3. Activate (Mac / Linux)
source env/bin/activate

# 3b. Activate (Windows PowerShell)

    pandas plotly pymupdf reportlab chromadb sentence-transformers \
    langgraph python-dotenv apscheduler twilio sendgrid folium \
    streamlit-folium fastapi uvicorn

# Deactivate when done

✅
Confirm it worked

Run which python — the path should point inside your env/ folder, not /usr/bin/python. If it does, you're isolated and ready.

Rewrite a Node.js Express Route in FastAPI
Side-by-side comparison. The mental model is nearly identical — the syntax is different.
🔄
The mental model maps 1-to-1

Express and FastAPI share the same decorator-based routing concept. app.get('/route', handler) in Express becomes @app.get('/route') above a function in FastAPI. The biggest difference: FastAPI uses Python type hints to automatically validate request data and generate Swagger docs.

Example: A route that accepts a shipment query, returns risk data. This mirrors your CargoSense idea.

Node.js + Express
// routes/shipment.js
const express = require('express');
const router = express.Router();

// GET /shipment/:id
router.get('/shipment/:id',
  async (req, res) => {
    const { id } = req.params;
    const { origin } = req.query;

    if (!id) {
      return res.status(400).json({
        error: 'id required'
      });
    }

    const data = await
      getShipmentData(id, origin);

    res.json({
      shipment_id: id,
      origin: origin,
      risk_score: data.risk,
      route: data.route
    });
  }
);

// POST /shipment/analyze
router.post('/shipment/analyze',
  async (req, res) => {
    const { cargo, route } = req.body;

    const result = await
      analyzeShipment(cargo, route);

    res.status(201).json({ result });
  }
);

module.exports = router;
Python + FastAPI
# routes/shipment.py
from fastapi import APIRouter, HTTPException
from pydantic import BaseModel
from typing import Optional

router = APIRouter()

# GET /shipment/{id}
@router.get("/shipment/{id}")
async def get_shipment(
    id: str,
    origin: Optional[str] = None
):
    if not id:
        raise HTTPException(
            status_code=400,
            detail="id required"
        )

    data = await get_shipment_data(id, origin)

    return {
        "shipment_id": id,
        "origin": origin,
        "risk_score": data.risk,
        "route": data.route
    }


# Request body model — replaces req.body
class AnalyzeRequest(BaseModel):
    cargo: str
    route: str

# POST /shipment/analyze
@router.post("/shipment/analyze",
             status_code=201)
async def analyze_shipment(
    body: AnalyzeRequest  # auto-parsed
):
    result = await analyze_shipment(
        body.cargo, body.route
    )
    return {"result": result}

main.py — wiring the router + running the server

from routes.shipment import router as shipment_router

03
Call Two APIs in Parallel with asyncio.gather()
Fire both requests simultaneously. Cut your wait time in half. Critical for your multi-agent ideas.
⚡
Why this matters for your hackathon

Your CargoSense, Supplier Reliability, and News Monitor agents all need to call multiple APIs (news, shipping, FX rates) before they can answer. Calling them sequentially means: API-1 wait + API-2 wait. With asyncio.gather(): just max(API-1, API-2). On a slow venue WiFi this can save 3–5 seconds per query — the difference between an impressive demo and a painfully slow one.

PATTERN — basic asyncio.gather() with httpx
import asyncio
import httpx

async def fetch_news(query: str) -> dict:
    """Calls NewsAPI — async, non-blocking."""
    async with httpx.AsyncClient() as client:
        response = await client.get(
            "https://newsapi.org/v2/everything",
            params={"q": query, "apiKey": "YOUR_KEY"}
        )
        return response.json()


async def fetch_exchange_rate(currency: str) -> dict:
    """Calls ExchangeRate-API — async, non-blocking."""
    async with httpx.AsyncClient() as client:
        response = await client.get(
            f"https://api.exchangerate-api.com/v4/latest/{currency}"
        )
        return response.json()


async def get_cargo_intelligence(
    query: str,
    currency: str = "USD"
) -> dict:
    """
    Fires BOTH calls at the same time.
    asyncio.gather() waits for all of them,
    then returns results in the same order.
    """
    news_data, fx_data = await asyncio.gather(
        fetch_news(query),
        fetch_exchange_rate(currency)
    )

    return {
        "news": news_data.get("articles", []),
        "pkr_rate": fx_data.get("rates", {}).get("PKR")
    }


# ── Run it (standalone script) ──
if __name__ == "__main__":
    result = asyncio.run(
        get_cargo_intelligence("Red Sea shipping Pakistan")
    )
    print(result)
ADVANCED — 3+ APIs in parallel with error handling
import asyncio, httpx
from typing import Any

async def safe_fetch(client: httpx.AsyncClient, url: str,
                      params: dict = None) -> Any:
    """Wrapper that catches errors so one failed API
       doesn't crash the others."""
    try:
        r = await client.get(url, params=params, timeout=8.0)
        return r.json()
    except Exception as e:
        return {"error": str(e)}


async def multi_agent_fetch(cargo_query: str) -> dict:
    """Calls 3 APIs in parallel — news, FX, shipping index."""
    async with httpx.AsyncClient() as client:
        news, fx, shipping = await asyncio.gather(
            safe_fetch(client,
                "https://newsapi.org/v2/everything",
                {"q": cargo_query, "apiKey": "KEY"}
            ),
            safe_fetch(client,
                "https://api.exchangerate-api.com/v4/latest/USD"
            ),
            safe_fetch(client,
                "https://data.nasdaq.com/api/v3/datasets/CHRIS/CME_CL1"
            ),
        )

    return {
        "news_articles": news.get("articles", [])[:5],
        "pkr_per_usd":   fx.get("rates", {}).get("PKR"),
        "shipping_index": shipping
    }


# ── Inside a FastAPI route ──
@app.get("/intelligence")
async def get_intelligence(cargo: str):
    # FastAPI is already async — just await it directly
    data = await multi_agent_fetch(cargo)
    return data

Load API Keys from .env with python-dotenv
Same pattern as Node.js dotenv. Never hardcode secrets. Git-safe from day one.
🔑
Same concept, different package name

In Node.js you use require('dotenv').config(). In Python you use from dotenv import load_dotenv; load_dotenv(). Both read a .env file and inject variables into the process environment. From that point, os.getenv('KEY') (Python) works exactly like process.env.KEY (Node.js).

Node.js pattern
// .env file (same format)
ANTHROPIC_API_KEY=sk-ant-...
NEWS_API_KEY=abc123
OPENWEATHER_KEY=xyz789

// index.js
require('dotenv').config();

const key = process.env.ANTHROPIC_API_KEY;
console.log(key);  // sk-ant-...
Python pattern
# .env file (IDENTICAL format)
# ANTHROPIC_API_KEY=sk-ant-...
# NEWS_API_KEY=abc123
# OPENWEATHER_KEY=xyz789

# config.py
import os
from dotenv import load_dotenv

load_dotenv()  # reads .env into env

key = os.getenv("ANTHROPIC_API_KEY")
print(key)  # sk-ant-...

Python Classes — __init__, Methods & Usage
Python classes are how you'll structure your agents, calculators, and data models at the hackathon.
🏗️
JS vs Python class syntax — the key differences

Python uses self where JS uses this. Python's constructor is __init__ (with double underscores) where JS uses constructor. Every instance method must take self as its first parameter. Python has no new keyword — you call the class like a function.

JavaScript class
class ShipmentAnalyzer {
  constructor(api_key, route) {
    this.api_key = api_key;
    this.route   = route;
    this.risk    = 0;
  }

  analyze(cargo) {
    const result = ...;
    this.risk = result.score;
    return result;
  }

  getRisk() {
    return this.risk;
  }
}

// Usage
const analyzer = new ShipmentAnalyzer(
  "sk-ant-...", "Suez"
);
analyzer.analyze("Electronics");
Python class
class ShipmentAnalyzer:
    def __init__(self, api_key: str,
                 route: str):
        self.api_key = api_key
        self.route   = route
        self.risk    = 0
        # No 'new', no {}, just self

    def analyze(self, cargo: str):
        result = ...
        self.risk = result["score"]
        return result

    def get_risk(self) -> int:
        return self.risk

# Usage — no 'new' keyword
analyzer = ShipmentAnalyzer(
    "sk-ant-...", "Suez"
)
analyzer.analyze("Electronics")
FULL EXAMPLE — CargoAgent class with __init__, instance methods, class methods, and async
import asyncio, os
import anthropic
from dotenv import load_dotenv

load_dotenv()


class CargoAgent:
    """
    Analyzes shipment risk for Pakistani importers.
    Demonstrates: __init__, instance methods, class
    methods, static methods, and async methods.
    """

    # Class variable (shared across ALL instances)
    SUPPORTED_ROUTES = ["Suez", "Hormuz", "Cape"]

    def __init__(self, route: str, cargo_type: str):
        """Constructor — runs when you create an instance."""
        self.route      = route       # instance variable
        self.cargo_type = cargo_type
        self.risk_score = 0
        self.analysis   = None
        self.client     = anthropic.Anthropic(
            api_key=os.getenv("ANTHROPIC_API_KEY")
        )

    # ── Instance method ──
    def describe(self) -> str:
        """Regular method — has access to self."""
        return (
            f"Route: {self.route} | "
            f"Cargo: {self.cargo_type} | "
            f"Risk: {self.risk_score}/10"
        )

    # ── Async instance method ──
    async def analyze(self, news_context: str) -> dict:
        """Calls Claude, stores result on self."""
        prompt = (
            f"Analyze shipment risk:\n"
            f"Route: {self.route}\n"
            f"Cargo: {self.cargo_type}\n"
            f"Context: {news_context}\n"
            "Return a risk score 1-10 and brief reason."
        )

        response = self.client.messages.create(
            model="claude-sonnet-4-20250514",
            max_tokens=300,
            messages=[{"role": "user", "content": prompt}]
        )

        self.analysis   = response.content[0].text
        self.risk_score = self._parse_score(self.analysis)
        return {"score": self.risk_score, "reason": self.analysis}

    # ── Private helper method (convention: _ prefix) ──
    def _parse_score(self, text: str) -> int:
        """Extract integer score from Claude's response."""
        for word in text.split():
            if word.isdigit() and 1 <= int(word) <= 10:
                return int(word)
        return 5  # default

    # ── Class method (operates on the class, not an instance) ──
    @classmethod
    def from_query(cls, query: str) -> "CargoAgent":
        """Alternative constructor — parses a text query."""
        route = "Suez"  # simplified parse
        for r in cls.SUPPORTED_ROUTES:
            if r.lower() in query.lower():
                route = r
        return cls(route=route, cargo_type="General")

    # ── Static method (utility — no self, no cls) ──
    @staticmethod
    def risk_label(score: int) -> str:
        if score >= 8:  return "🔴 HIGH"
        if score >= 5:  return "🟡 MEDIUM"
        return "🟢 LOW"


# ──────────────────────────────────────────────
# USAGE
# ──────────────────────────────────────────────
async def main():
    # Standard construction
    agent = CargoAgent(route="Hormuz", cargo_type="Electronics")

    # Alternative constructor via classmethod
    agent2 = CargoAgent.from_query("Suez route textiles")

    # Call async method
    result = await agent.analyze("US/Iran tensions escalating")

    # Instance method
    print(agent.describe())

    # Static method (no instance needed)
    label = CargoAgent.risk_label(result["score"])
    print(f"Risk level: {label}")


if __name__ == "__main__":
    asyncio.run(main())

🧠
Which class type should you use at the hackathon?

For your agent logic (CargoAgent, SupplierChecker, FXAdvisor): plain Python class with __init__ + async methods. For request/response data models: Pydantic's BaseModel (which is also a Python class). Pydantic models auto-validate, auto-document, and auto-serialize to JSON — use them for all FastAPI endpoints.

PYDANTIC BaseModel — the class type you'll use most in FastAPI
from pydantic import BaseModel
from typing import Optional
from fastapi import FastAPI

app = FastAPI()

# Define your data model — Python class inheriting BaseModel
class ShipmentRequest(BaseModel):
    cargo_type: str
    route: str
    value_usd: float
    currency: Optional[str] = "USD"   # default value

class RiskResponse(BaseModel):
    risk_score: int
    label: str
    recommendation: str
    estimated_cost_pkr: float

# FastAPI reads ShipmentRequest from the POST body automatically
@app.post("/analyze", response_model=RiskResponse)
async def analyze_shipment(req: ShipmentRequest):
    # req.cargo_type, req.route, req.value_usd — all type-safe
    agent = CargoAgent(req.route, req.cargo_type)
    result = await agent.analyze("context here")

    return RiskResponse(
        risk_score=result["score"],
        label=CargoAgent.risk_label(result["score"]),
        recommendation=result["reason"],
        estimated_cost_pkr=req.value_usd * 278.5
    )
